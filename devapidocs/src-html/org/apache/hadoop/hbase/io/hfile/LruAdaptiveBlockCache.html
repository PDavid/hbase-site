050/**
051 * <b>This realisation improve performance of classical LRU
052 * cache up to 3 times via reduce GC job.</b>
053 * </p>
054 * The classical block cache implementation that is memory-aware using {@link HeapSize},
055 * memory-bound using an
056 * LRU eviction algorithm, and concurrent: backed by a {@link ConcurrentHashMap} and with a
057 * non-blocking eviction thread giving constant-time {@link #cacheBlock} and {@link #getBlock}
058 * operations.
059 * </p>
060 * Contains three levels of block priority to allow for scan-resistance and in-memory families
061 * {@link org.apache.hadoop.hbase.client.ColumnFamilyDescriptorBuilder#setInMemory(boolean)} (An
062 * in-memory column family is a column family that should be served from memory if possible):
063 * single-access, multiple-accesses, and in-memory priority. A block is added with an in-memory
064 * priority flag if {@link org.apache.hadoop.hbase.client.ColumnFamilyDescriptor#isInMemory()},
065 * otherwise a block becomes a single access priority the first time it is read into this block
066 * cache. If a block is accessed again while in cache, it is marked as a multiple access priority
067 * block. This delineation of blocks is used to prevent scans from thrashing the cache adding a
068 * least-frequently-used element to the eviction algorithm.
069 * <p/>
070 * Each priority is given its own chunk of the total cache to ensure fairness during eviction. Each
071 * priority will retain close to its maximum size, however, if any priority is not using its entire
072 * chunk the others are able to grow beyond their chunk size.
073 * <p/>
074 * Instantiated at a minimum with the total size and average block size. All sizes are in bytes. The
075 * block size is not especially important as this cache is fully dynamic in its sizing of blocks. It
076 * is only used for pre-allocating data structures and in initial heap estimation of the map.
077 * <p/>
078 * The detailed constructor defines the sizes for the three priorities (they should total to the
079 * <code>maximum size</code> defined). It also sets the levels that trigger and control the eviction
080 * thread.
081 * <p/>
082 * The <code>acceptable size</code> is the cache size level which triggers the eviction process to
083 * start. It evicts enough blocks to get the size below the minimum size specified.
084 * <p/>
085 * Eviction happens in a separate thread and involves a single full-scan of the map. It determines
086 * how many bytes must be freed to reach the minimum size, and then while scanning determines the
087 * fewest least-recently-used blocks necessary from each of the three priorities (would be 3 times
088 * bytes to free). It then uses the priority chunk sizes to evict fairly according to the relative
089 * sizes and usage.
090 * <p/>
091 * Adaptive LRU cache lets speed up performance while we are reading much more data than can fit
092 * into BlockCache and it is the cause of a high rate of evictions. This in turn leads to heavy
093 * Garbage Collector works. So a lot of blocks put into BlockCache but never read, but spending
094 * a lot of CPU resources for cleaning. We could avoid this situation via parameters:
095 * <p/>
096 * <b>hbase.lru.cache.heavy.eviction.count.limit</b>  - set how many times we have to run the
097 * eviction process that starts to avoid putting data to BlockCache. By default it is 0 and it
098 * meats the feature will start at the beginning. But if we have some times short reading the same
099 * data and some times long-term reading - we can divide it by this parameter. For example we know
100 * that our short reading used to be about 1 minutes, then we have to set the parameter about 10
101 * and it will enable the feature only for long time massive reading (after ~100 seconds). So when
102 * we use short-reading and want all of them in the cache we will have it (except for eviction of
103 * course). When we use long-term heavy reading the feature will be enabled after some time and
104 * bring better performance.
105 * <p/>
106 * <b>hbase.lru.cache.heavy.eviction.mb.size.limit</b> - set how many bytes in 10 seconds desirable
107 * putting into BlockCache (and evicted from it). The feature will try to reach this value and
108 * maintain it. Don't try to set it too small because it leads to premature exit from this mode.
109 * For powerful CPUs (about 20-40 physical cores)  it could be about 400-500 MB. Average system
110 * (~10 cores) 200-300 MB. Some weak systems (2-5 cores) may be good with 50-100 MB.
111 * How it works: we set the limit and after each ~10 second calculate how many bytes were freed.
112 * Overhead = Freed Bytes Sum (MB) * 100 / Limit (MB) - 100;
113 * For example we set the limit = 500 and were evicted 2000 MB. Overhead is:
114 * 2000 * 100 / 500 - 100 = 300%
115 * The feature is going to reduce a percent caching data blocks and fit evicted bytes closer to
116 * 100% (500 MB). Some kind of an auto-scaling.
117 * If freed bytes less then the limit we have got negative overhead.
118 * For example if were freed 200 MB:
119 * 200 * 100 / 500 - 100 = -60%
120 * The feature will increase the percent of caching blocks.
121 * That leads to fit evicted bytes closer to 100% (500 MB).
122 * The current situation we can find out in the log of RegionServer:
123 * BlockCache evicted (MB): 0, overhead (%): -100, heavy eviction counter: 0, current caching
124 * DataBlock (%): 100 - means no eviction, 100% blocks is caching
125 * BlockCache evicted (MB): 2000, overhead (%): 300, heavy eviction counter: 1, current caching
126 * DataBlock (%): 97 - means eviction begin, reduce of caching blocks by 3%.
127 * It help to tune your system and find out what value is better set. Don't try to reach 0%
128 * overhead, it is impossible. Quite good 50-100% overhead,
129 * it prevents premature exit from this mode.
130 * <p/>
131 * <b>hbase.lru.cache.heavy.eviction.overhead.coefficient</b> - set how fast we want to get the
132 * result. If we know that our reading is heavy for a long time, we don't want to wait and can
133 * increase the coefficient and get good performance sooner. But if we aren't sure we can do it
134 * slowly and it could prevent premature exit from this mode. So, when the coefficient is higher
135 * we can get better performance when heavy reading is stable. But when reading is changing we
136 * can adjust to it and set  the coefficient to lower value.
137 * For example, we set the coefficient = 0.01. It means the overhead (see above) will be
138 * multiplied by 0.01 and the result is the value of reducing percent caching blocks. For example,
139 * if the overhead = 300% and the coefficient = 0.01,
140 * then percent of caching blocks will reduce by 3%.
141 * Similar logic when overhead has got negative value (overshooting).  Maybe it is just short-term
142 * fluctuation and we will try to stay in this mode. It helps avoid premature exit during
143 * short-term fluctuation. Backpressure has simple logic: more overshooting - more caching blocks.
144 * <p/>
145 * Find more information about improvement: https://issues.apache.org/jira/browse/HBASE-23887
146 */

384   * @param mapLoadFactor       initial load factor of backing ConcurrentHashMap
385   * @param mapConcurrencyLevel initial concurrency factor for backing CHM
386   * @param minFactor           percentage of total size that eviction will evict until
387   * @param acceptableFactor    percentage of total size that triggers eviction
388   * @param singleFactor        percentage of total size for single-access blocks
389   * @param multiFactor         percentage of total size for multiple-access blocks
390   * @param memoryFactor        percentage of total size for in-memory blocks
391   * @param hardLimitFactor     hard capacity limit
392   * @param forceInMemory       in-memory hfile's data block has higher priority when evicting
393   * @param maxBlockSize        maximum block size for caching
394   * @param heavyEvictionCountLimit   when starts AdaptiveLRU algoritm work
395   * @param heavyEvictionMbSizeLimit  how many bytes desirable putting into BlockCache
396   * @param heavyEvictionOverheadCoefficient  how aggressive AdaptiveLRU will reduce GC
397   */
398  public LruAdaptiveBlockCache(long maxSize, long blockSize, boolean evictionThread,
399    int mapInitialSize, float mapLoadFactor, int mapConcurrencyLevel,
400    float minFactor, float acceptableFactor, float singleFactor,
401    float multiFactor, float memoryFactor, float hardLimitFactor,
402    boolean forceInMemory, long maxBlockSize,
403    int heavyEvictionCountLimit, long heavyEvictionMbSizeLimit,
404    float heavyEvictionOverheadCoefficient) {
405    this.maxBlockSize = maxBlockSize;
406    if(singleFactor + multiFactor + memoryFactor != 1 ||
407      singleFactor < 0 || multiFactor < 0 || memoryFactor < 0) {
408      throw new IllegalArgumentException("Single, multi, and memory factors " +
409        " should be non-negative and total 1.0");
410    }
411    if (minFactor >= acceptableFactor) {
412      throw new IllegalArgumentException("minFactor must be smaller than acceptableFactor");
413    }
414    if (minFactor >= 1.0f || acceptableFactor >= 1.0f) {
415      throw new IllegalArgumentException("all factors must be < 1");
416    }
417    this.maxSize = maxSize;
418    this.blockSize = blockSize;
419    this.forceInMemory = forceInMemory;
420    map = new ConcurrentHashMap<>(mapInitialSize, mapLoadFactor, mapConcurrencyLevel);
421    this.minFactor = minFactor;
422    this.acceptableFactor = acceptableFactor;
423    this.singleFactor = singleFactor;
424    this.multiFactor = multiFactor;
425    this.memoryFactor = memoryFactor;
426    this.stats = new CacheStats(this.getClass().getSimpleName());
427    this.count = new AtomicLong(0);
428    this.elements = new AtomicLong(0);
429    this.dataBlockElements = new LongAdder();
430    this.dataBlockSize = new LongAdder();
431    this.overhead = calculateOverhead(maxSize, blockSize, mapConcurrencyLevel);
432    this.size = new AtomicLong(this.overhead);
433    this.hardCapacityLimitFactor = hardLimitFactor;
434    if (evictionThread) {
435      this.evictionThread = new EvictionThread(this);
436      this.evictionThread.start(); // FindBugs SC_START_IN_CTOR
437    } else {
438      this.evictionThread = null;
439    }
440
441    // check the bounds
442    this.heavyEvictionCountLimit = Math.max(heavyEvictionCountLimit, 0);
443    this.heavyEvictionMbSizeLimit = Math.max(heavyEvictionMbSizeLimit, 1);
444    this.cacheDataBlockPercent = 100;
445    heavyEvictionOverheadCoefficient = Math.min(heavyEvictionOverheadCoefficient, 1.0f);
446    heavyEvictionOverheadCoefficient = Math.max(heavyEvictionOverheadCoefficient, 0.001f);
447    this.heavyEvictionOverheadCoefficient = heavyEvictionOverheadCoefficient;
448
449    // TODO: Add means of turning this off.  Bit obnoxious running thread just to make a log
450    // every five minutes.
451    this.scheduleThreadPool.scheduleAtFixedRate(new StatisticsThread(this), STAT_THREAD_PERIOD,
452      STAT_THREAD_PERIOD, TimeUnit.SECONDS);
453  }

475  /**
476   * The block cached in LruAdaptiveBlockCache will always be an heap block: on the one side,
477   * the heap access will be more faster then off-heap, the small index block or meta block
478   * cached in CombinedBlockCache will benefit a lot. on other side, the LruAdaptiveBlockCache
479   * size is always calculated based on the total heap size, if caching an off-heap block in
480   * LruAdaptiveBlockCache, the heap size will be messed up. Here we will clone the block into an
481   * heap block if it's an off-heap block, otherwise just use the original block. The key point is
482   * maintain the refCnt of the block (HBASE-22127): <br>
483   * 1. if cache the cloned heap block, its refCnt is an totally new one, it's easy to handle; <br>
484   * 2. if cache the original heap block, we're sure that it won't be tracked in ByteBuffAllocator's
485   * reservoir, if both RPC and LruAdaptiveBlockCache release the block, then it can be garbage
486   * collected by JVM, so need a retain here.
487   * @param buf the original block
488   * @return an block with an heap memory backend.
489   */
490  private Cacheable asReferencedHeapBlock(Cacheable buf) {
491    if (buf instanceof HFileBlock) {
492      HFileBlock blk = ((HFileBlock) buf);
493      if (blk.isSharedMem()) {
494        return HFileBlock.deepCloneOnHeap(blk);
495      }
496    }
497    // The block will be referenced by this LruAdaptiveBlockCache,
498    // so should increase its refCnt here.
499    return buf.retain();
500  }

504  /**
505   * Cache the block with the specified name and buffer.
506   * <p>
507   * It is assumed this will NOT be called on an already cached block. In rare cases (HBASE-8547)
508   * this can happen, for which we compare the buffer contents.
509   *
510   * @param cacheKey block's cache key
511   * @param buf      block buffer
512   * @param inMemory if block is in-memory
513   */
514  @Override
515  public void cacheBlock(BlockCacheKey cacheKey, Cacheable buf, boolean inMemory) {
516
517    // Some data blocks will not put into BlockCache when eviction rate too much.
518    // It is good for performance
519    // (see details: https://issues.apache.org/jira/browse/HBASE-23887)
520    // How to calculate it can find inside EvictionThread class.
521    if (cacheDataBlockPercent != 100 && buf.getBlockType().isData()) {
522      // It works like filter - blocks which two last digits of offset
523      // more than we calculate in Eviction Thread will not put into BlockCache
524      if (cacheKey.getOffset() % 100 >= cacheDataBlockPercent) {
525        return;
526      }
527    }
528
529    if (buf.heapSize() > maxBlockSize) {
530      // If there are a lot of blocks that are too
531      // big this can make the logs way too noisy.
532      // So we log 2%
533      if (stats.failInsert() % 50 == 0) {
534        LOG.warn("Trying to cache too large a block "
535          + cacheKey.getHfileName() + " @ "
536          + cacheKey.getOffset()
537          + " is " + buf.heapSize()
538          + " which is larger than " + maxBlockSize);
539      }
540      return;
541    }
542
543    LruCachedBlock cb = map.get(cacheKey);
544    if (cb != null && !BlockCacheUtil.shouldReplaceExistingCacheBlock(this,
545      cacheKey, buf)) {
546      return;
547    }
548    long currentSize = size.get();
549    long currentAcceptableSize = acceptableSize();
550    long hardLimitSize = (long) (hardCapacityLimitFactor * currentAcceptableSize);
551    if (currentSize >= hardLimitSize) {
552      stats.failInsert();
553      if (LOG.isTraceEnabled()) {
554        LOG.trace("LruAdaptiveBlockCache current size " + StringUtils.byteDesc(currentSize)
555          + " has exceeded acceptable size " + StringUtils.byteDesc(currentAcceptableSize) + "."
556          + " The hard limit size is " + StringUtils.byteDesc(hardLimitSize)
557          + ", failed to put cacheKey:" + cacheKey + " into LruAdaptiveBlockCache.");
558      }
559      if (!evictionInProgress) {
560        runEviction();
561      }
562      return;
563    }
564    // Ensure that the block is an heap one.
565    buf = asReferencedHeapBlock(buf);
566    cb = new LruCachedBlock(cacheKey, buf, count.incrementAndGet(), inMemory);
567    long newSize = updateSizeMetrics(cb, false);
568    map.put(cacheKey, cb);
569    long val = elements.incrementAndGet();
570    if (buf.getBlockType().isData()) {
571      dataBlockElements.increment();
572    }
573    if (LOG.isTraceEnabled()) {
574      long size = map.size();
575      assertCounterSanity(size, val);
576    }
577    if (newSize > currentAcceptableSize && !evictionInProgress) {
578      runEviction();
579    }
580  }

582  /**
583   * Sanity-checking for parity between actual block cache content and metrics.
584   * Intended only for use with TRACE level logging and -ea JVM.
585   */
586  private static void assertCounterSanity(long mapSize, long counterVal) {
587    if (counterVal < 0) {
588      LOG.trace("counterVal overflow. Assertions unreliable. counterVal=" + counterVal +
589        ", mapSize=" + mapSize);
590      return;
591    }
592    if (mapSize < Integer.MAX_VALUE) {
593      double pct_diff = Math.abs((((double) counterVal) / ((double) mapSize)) - 1.);
594      if (pct_diff > 0.05) {
595        LOG.trace("delta between reported and actual size > 5%. counterVal=" + counterVal +
596          ", mapSize=" + mapSize);
597      }
598    }
599  }

633  /**
634   * Get the buffer of the block with the specified name.
635   *
636   * @param cacheKey           block's cache key
637   * @param caching            true if the caller caches blocks on cache misses
638   * @param repeat             Whether this is a repeat lookup for the same block
639   *                           (used to avoid double counting cache misses when doing double-check
640   *                           locking)
641   * @param updateCacheMetrics Whether to update cache metrics or not
642   *
643   * @return buffer of specified cache key, or null if not in cache
644   */
645  @Override
646  public Cacheable getBlock(BlockCacheKey cacheKey, boolean caching, boolean repeat,
647    boolean updateCacheMetrics) {
648    LruCachedBlock cb = map.computeIfPresent(cacheKey, (key, val) -> {
649      // It will be referenced by RPC path, so increase here. NOTICE: Must do the retain inside
650      // this block. because if retain outside the map#computeIfPresent, the evictBlock may remove
651      // the block and release, then we're retaining a block with refCnt=0 which is disallowed.
652      // see HBASE-22422.
653      val.getBuffer().retain();
654      return val;
655    });
656    if (cb == null) {
657      if (!repeat && updateCacheMetrics) {
658        stats.miss(caching, cacheKey.isPrimary(), cacheKey.getBlockType());
659      }
660      // If there is another block cache then try and read there.
661      // However if this is a retry ( second time in double checked locking )
662      // And it's already a miss then the l2 will also be a miss.
663      if (victimHandler != null && !repeat) {
664        // The handler will increase result's refCnt for RPC, so need no extra retain.
665        Cacheable result = victimHandler.getBlock(cacheKey, caching, repeat, updateCacheMetrics);
666        // Promote this to L1.
667        if (result != null) {
668          if (caching) {
669            cacheBlock(cacheKey, result, /* inMemory = */ false);
670          }
671        }
672        return result;
673      }
674      return null;
675    }
676    if (updateCacheMetrics) {
677      stats.hit(caching, cacheKey.isPrimary(), cacheKey.getBlockType());
678    }
679    cb.access(count.incrementAndGet());
680    return cb.getBuffer();
681  }

719  /**
720   * Evict the block, and it will be cached by the victim handler if exists &amp;&amp;
721   * block may be read again later
722   *
723   * @param evictedByEvictionProcess true if the given block is evicted by
724   *          EvictionThread
725   * @return the heap size of evicted block
726   */
727  protected long evictBlock(LruCachedBlock block, boolean evictedByEvictionProcess) {
728    LruCachedBlock previous = map.remove(block.getCacheKey());
729    if (previous == null) {
730      return 0;
731    }
732    updateSizeMetrics(block, true);
733    long val = elements.decrementAndGet();
734    if (LOG.isTraceEnabled()) {
735      long size = map.size();
736      assertCounterSanity(size, val);
737    }
738    if (block.getBuffer().getBlockType().isData()) {
739      dataBlockElements.decrement();
740    }
741    if (evictedByEvictionProcess) {
742      // When the eviction of the block happened because of invalidation of HFiles, no need to
743      // update the stats counter.
744      stats.evicted(block.getCachedTime(), block.getCacheKey().isPrimary());
745      if (victimHandler != null) {
746        victimHandler.cacheBlock(block.getCacheKey(), block.getBuffer());
747      }
748    }
749    // Decrease the block's reference count, and if refCount is 0, then it'll auto-deallocate. DO
750    // NOT move this up because if do that then the victimHandler may access the buffer with
751    // refCnt = 0 which is disallowed.
752    previous.getBuffer().release();
753    return block.heapSize();
754  }

775  /**
776   * Eviction method.
777   *
778   * Evict items in order of use, allowing delete items
779   * which haven't been used for the longest amount of time.
780   *
781   * @return how many bytes were freed
782   */
783  long evict() {
784
785    // Ensure only one eviction at a time
786    if (!evictionLock.tryLock()) {
787      return 0;
788    }
789
790    long bytesToFree = 0L;
791
792    try {
793      evictionInProgress = true;
794      long currentSize = this.size.get();
795      bytesToFree = currentSize - minSize();
796
797      if (LOG.isTraceEnabled()) {
798        LOG.trace("Block cache LRU eviction started; Attempting to free " +
799          StringUtils.byteDesc(bytesToFree) + " of total=" +
800          StringUtils.byteDesc(currentSize));
801      }
802
803      if (bytesToFree <= 0) {
804        return 0;
805      }
806
807      // Instantiate priority buckets
808      BlockBucket bucketSingle
809        = new BlockBucket("single", bytesToFree, blockSize, singleSize());
810      BlockBucket bucketMulti
811        = new BlockBucket("multi", bytesToFree, blockSize, multiSize());
812      BlockBucket bucketMemory
813        = new BlockBucket("memory", bytesToFree, blockSize, memorySize());
814
815      // Scan entire map putting into appropriate buckets
816      for (LruCachedBlock cachedBlock : map.values()) {
817        switch (cachedBlock.getPriority()) {
818          case SINGLE: {
819            bucketSingle.add(cachedBlock);
820            break;
821          }
822          case MULTI: {
823            bucketMulti.add(cachedBlock);
824            break;
825          }
826          case MEMORY: {
827            bucketMemory.add(cachedBlock);
828            break;
829          }
830        }
831      }
832
833      long bytesFreed = 0;
834      if (forceInMemory || memoryFactor > 0.999f) {
835        long s = bucketSingle.totalSize();
836        long m = bucketMulti.totalSize();
837        if (bytesToFree > (s + m)) {
838          // this means we need to evict blocks in memory bucket to make room,
839          // so the single and multi buckets will be emptied
840          bytesFreed = bucketSingle.free(s);
841          bytesFreed += bucketMulti.free(m);
842          if (LOG.isTraceEnabled()) {
843            LOG.trace("freed " + StringUtils.byteDesc(bytesFreed) +
844              " from single and multi buckets");
845          }
846          bytesFreed += bucketMemory.free(bytesToFree - bytesFreed);
847          if (LOG.isTraceEnabled()) {
848            LOG.trace("freed " + StringUtils.byteDesc(bytesFreed) +
849              " total from all three buckets ");
850          }
851        } else {
852          // this means no need to evict block in memory bucket,
853          // and we try best to make the ratio between single-bucket and
854          // multi-bucket is 1:2
855          long bytesRemain = s + m - bytesToFree;
856          if (3 * s <= bytesRemain) {
857            // single-bucket is small enough that no eviction happens for it
858            // hence all eviction goes from multi-bucket
859            bytesFreed = bucketMulti.free(bytesToFree);
860          } else if (3 * m <= 2 * bytesRemain) {
861            // multi-bucket is small enough that no eviction happens for it
862            // hence all eviction goes from single-bucket
863            bytesFreed = bucketSingle.free(bytesToFree);
864          } else {
865            // both buckets need to evict some blocks
866            bytesFreed = bucketSingle.free(s - bytesRemain / 3);
867            if (bytesFreed < bytesToFree) {
868              bytesFreed += bucketMulti.free(bytesToFree - bytesFreed);
869            }
870          }
871        }
872      } else {
873        PriorityQueue<BlockBucket> bucketQueue = new PriorityQueue<>(3);
874
875        bucketQueue.add(bucketSingle);
876        bucketQueue.add(bucketMulti);
877        bucketQueue.add(bucketMemory);
878
879        int remainingBuckets = bucketQueue.size();
880
881        BlockBucket bucket;
882        while ((bucket = bucketQueue.poll()) != null) {
883          long overflow = bucket.overflow();
884          if (overflow > 0) {
885            long bucketBytesToFree =
886              Math.min(overflow, (bytesToFree - bytesFreed) / remainingBuckets);
887            bytesFreed += bucket.free(bucketBytesToFree);
888          }
889          remainingBuckets--;
890        }
891      }
892      if (LOG.isTraceEnabled()) {
893        long single = bucketSingle.totalSize();
894        long multi = bucketMulti.totalSize();
895        long memory = bucketMemory.totalSize();
896        LOG.trace("Block cache LRU eviction completed; " +
897          "freed=" + StringUtils.byteDesc(bytesFreed) + ", " +
898          "total=" + StringUtils.byteDesc(this.size.get()) + ", " +
899          "single=" + StringUtils.byteDesc(single) + ", " +
900          "multi=" + StringUtils.byteDesc(multi) + ", " +
901          "memory=" + StringUtils.byteDesc(memory));
902      }
903    } finally {
904      stats.evict();
905      evictionInProgress = false;
906      evictionLock.unlock();
907    }
908    return bytesToFree;
909  }

928  /**
929   * Used to group blocks into priority buckets.  There will be a BlockBucket
930   * for each priority (single, multi, memory).  Once bucketed, the eviction
931   * algorithm takes the appropriate number of elements out of each according
932   * to configuration parameters and their relatives sizes.
933   */
934  private class BlockBucket implements Comparable<BlockBucket> {
935
936    private final String name;
937    private final LruCachedBlockQueue queue;
938    private long totalSize = 0;
939    private final long bucketSize;
940
941    public BlockBucket(String name, long bytesToFree, long blockSize, long bucketSize) {
942      this.name = name;
943      this.bucketSize = bucketSize;
944      queue = new LruCachedBlockQueue(bytesToFree, blockSize);
945      totalSize = 0;
946    }
947
948    public void add(LruCachedBlock block) {
949      totalSize += block.heapSize();
950      queue.add(block);
951    }
952
953    public long free(long toFree) {
954      if (LOG.isTraceEnabled()) {
955        LOG.trace("freeing {} from {}", StringUtils.byteDesc(toFree), this);
956      }
957      LruCachedBlock cb;
958      long freedBytes = 0;
959      while ((cb = queue.pollLast()) != null) {
960        freedBytes += evictBlock(cb, true);
961        if (freedBytes >= toFree) {
962          return freedBytes;
963        }
964      }
965      if (LOG.isTraceEnabled()) {
966        LOG.trace("freeing {} from {}", StringUtils.byteDesc(toFree), this);
967      }
968      return freedBytes;
969    }
970
971    public long overflow() {
972      return totalSize - bucketSize;
973    }
974
975    public long totalSize() {
976      return totalSize;
977    }

979    @Override
980    public int compareTo(BlockBucket that) {
981      return Long.compare(this.overflow(), that.overflow());
982    }
983
984    @Override
985    public boolean equals(Object that) {
986      if (!(that instanceof BlockBucket)) {
987        return false;
988      }
989      return compareTo((BlockBucket)that) == 0;
990    }

992    @Override
993    public int hashCode() {
994      return Objects.hashCode(name, bucketSize, queue, totalSize);
995    }

997    @Override
998    public String toString() {
999      return MoreObjects.toStringHelper(this)
1000        .add("name", name)
1001        .add("totalSize", StringUtils.byteDesc(totalSize))
1002        .add("bucketSize", StringUtils.byteDesc(bucketSize))
1003        .toString();
1004    }

1052  /*
1053   * Eviction thread.  Sits in waiting state until an eviction is triggered
1054   * when the cache size grows above the acceptable level.<p>
1055   *
1056   * Thread is triggered into action by {@link LruAdaptiveBlockCache#runEviction()}
1057   */
1058  static class EvictionThread extends Thread {
1059
1060    private WeakReference<LruAdaptiveBlockCache> cache;
1061    private volatile boolean go = true;
1062    // flag set after enter the run method, used for test
1063    private boolean enteringRun = false;
1064
1065    public EvictionThread(LruAdaptiveBlockCache cache) {
1066      super(Thread.currentThread().getName() + ".LruAdaptiveBlockCache.EvictionThread");
1067      setDaemon(true);
1068      this.cache = new WeakReference<>(cache);
1069    }
1070
1071    @Override
1072    public void run() {
1073      enteringRun = true;
1074      long freedSumMb = 0;
1075      int heavyEvictionCount = 0;
1076      int freedDataOverheadPercent = 0;
1077      long startTime = System.currentTimeMillis();
1078      while (this.go) {
1079        synchronized (this) {
1080          try {
1081            this.wait(1000 * 10/*Don't wait for ever*/);
1082          } catch (InterruptedException e) {
1083            LOG.warn("Interrupted eviction thread ", e);
1084            Thread.currentThread().interrupt();
1085          }
1086        }
1087        LruAdaptiveBlockCache cache = this.cache.get();
1088        if (cache == null) {
1089          break;
1090        }
1091        freedSumMb += cache.evict()/1024/1024;
1092        /*
1093         * Sometimes we are reading more data than can fit into BlockCache
1094         * and it is the cause a high rate of evictions.
1095         * This in turn leads to heavy Garbage Collector works.
1096         * So a lot of blocks put into BlockCache but never read,
1097         * but spending a lot of CPU resources.
1098         * Here we will analyze how many bytes were freed and decide
1099         * decide whether the time has come to reduce amount of caching blocks.
1100         * It help avoid put too many blocks into BlockCache
1101         * when evict() works very active and save CPU for other jobs.
1102         * More delails: https://issues.apache.org/jira/browse/HBASE-23887
1103         */
1104
1105        // First of all we have to control how much time
1106        // has passed since previuos evict() was launched
1107        // This is should be almost the same time (+/- 10s)
1108        // because we get comparable volumes of freed bytes each time.
1109        // 10s because this is default period to run evict() (see above this.wait)
1110        long stopTime = System.currentTimeMillis();
1111        if ((stopTime - startTime) > 1000 * 10 - 1) {
1112          // Here we have to calc what situation we have got.
1113          // We have the limit "hbase.lru.cache.heavy.eviction.bytes.size.limit"
1114          // and can calculte overhead on it.
1115          // We will use this information to decide,
1116          // how to change percent of caching blocks.
1117          freedDataOverheadPercent =
1118            (int) (freedSumMb * 100 / cache.heavyEvictionMbSizeLimit) - 100;
1119          if (freedSumMb > cache.heavyEvictionMbSizeLimit) {
1120            // Now we are in the situation when we are above the limit
1121            // But maybe we are going to ignore it because it will end quite soon
1122            heavyEvictionCount++;
1123            if (heavyEvictionCount > cache.heavyEvictionCountLimit) {
1124              // It is going for a long time and we have to reduce of caching
1125              // blocks now. So we calculate here how many blocks we want to skip.
1126              // It depends on:
1127              // 1. Overhead - if overhead is big we could more aggressive
1128              // reducing amount of caching blocks.
1129              // 2. How fast we want to get the result. If we know that our
1130              // heavy reading for a long time, we don't want to wait and can
1131              // increase the coefficient and get good performance quite soon.
1132              // But if we don't sure we can do it slowly and it could prevent
1133              // premature exit from this mode. So, when the coefficient is
1134              // higher we can get better performance when heavy reading is stable.
1135              // But when reading is changing we can adjust to it and set
1136              // the coefficient to lower value.
1137              int change =
1138                (int) (freedDataOverheadPercent * cache.heavyEvictionOverheadCoefficient);
1139              // But practice shows that 15% of reducing is quite enough.
1140              // We are not greedy (it could lead to premature exit).
1141              change = Math.min(15, change);
1142              change = Math.max(0, change); // I think it will never happen but check for sure
1143              // So this is the key point, here we are reducing % of caching blocks
1144              cache.cacheDataBlockPercent -= change;
1145              // If we go down too deep we have to stop here, 1% any way should be.
1146              cache.cacheDataBlockPercent = Math.max(1, cache.cacheDataBlockPercent);
1147            }
1148          } else {
1149            // Well, we have got overshooting.
1150            // Mayby it is just short-term fluctuation and we can stay in this mode.
1151            // It help avoid permature exit during short-term fluctuation.
1152            // If overshooting less than 90%, we will try to increase the percent of
1153            // caching blocks and hope it is enough.
1154            if (freedSumMb >= cache.heavyEvictionMbSizeLimit * 0.1) {
1155              // Simple logic: more overshooting - more caching blocks (backpressure)
1156              int change = (int) (-freedDataOverheadPercent * 0.1 + 1);
1157              cache.cacheDataBlockPercent += change;
1158              // But it can't be more then 100%, so check it.
1159              cache.cacheDataBlockPercent = Math.min(100, cache.cacheDataBlockPercent);
1160            } else {
1161              // Looks like heavy reading is over.
1162              // Just exit form this mode.
1163              heavyEvictionCount = 0;
1164              cache.cacheDataBlockPercent = 100;
1165            }
1166          }
1167          LOG.info("BlockCache evicted (MB): {}, overhead (%): {}, " +
1168              "heavy eviction counter: {}, " +
1169              "current caching DataBlock (%): {}",
1170            freedSumMb, freedDataOverheadPercent,
1171            heavyEvictionCount, cache.cacheDataBlockPercent);
1172
1173          freedSumMb = 0;
1174          startTime = stopTime;
1175        }
1176      }
1177    }

1193     * Used for the test.

1375  @Override
1376  public void shutdown() {
1377    if (victimHandler != null) {
1378      victimHandler.shutdown();
1379    }
1380    this.scheduleThreadPool.shutdown();
1381    for (int i = 0; i < 10; i++) {
1382      if (!this.scheduleThreadPool.isShutdown()) {
1383        try {
1384          Thread.sleep(10);
1385        } catch (InterruptedException e) {
1386          LOG.warn("Interrupted while sleeping");
1387          Thread.currentThread().interrupt();
1388          break;
1389        }
1390      }
1391    }
1392
1393    if (!this.scheduleThreadPool.isShutdown()) {
1394      List<Runnable> runnables = this.scheduleThreadPool.shutdownNow();
1395      LOG.debug("Still running " + runnables);
1396    }
1397    this.evictionThread.shutdown();
1398  }
